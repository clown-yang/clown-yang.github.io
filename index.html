MySQL数据库查询总结

2018-09-09

查询

单表查询

查询语句的格式

select 字段1，字段2，字段3，....，字段n         6
from 表名                 1
where 条件子句            2
group by 分组子句         3
having 条件子句           4
order by 排序子句         5
limit 分页子句

例如：select * from person
该语句中，*代表了表中所有的字段，但是我们日常开发中，不允许这样使用。一定要明确的标出需要查询的相关字段名。这有有利于在查询时减少数据库在查询表时不必要的字段检索。

关系运算与逻辑运算
逻辑与 and
select pno, pname, age, hire_date, salary from persons where hire_date>='2018-03-01' and hire_date<='2018-04-30';
等价于
select pno, pname, age, hire_date, salary from persons where hire_date between '2018-03-01' and '2018-04-30';
第二种写法的效率会更优。因为采用了关系表达式，使用一次比较实现条件。

逻辑或 or
select pno, pname, age, hire_date, salary from persons where salary=3459 or salary = 3400;
等价于
select pno, pname, age, hire_date, salary from persons where salary in(3459,3400);

逻辑非 not
select pno, pname, age, hire_date, salary from persons where hire_date NOT between '2018-03-01' and '2018-04-30';

排序子句：order by（排序子句一定要放在所有子句之后）
sql中排序分为升序与降序，默认情况下为升序，如果要使用降序采用desc关键字。
select pno, pname, age, hire_date, salary from persons order by salary desc;
注意，如果出现多个排序条件时，表示为先按照第一个排序条件进行排序，如果出现相同的排序字段内容时，再按照第二个排序条件进行排序，依次类推。多个排序条件之间，用,号隔开。
select pno, pname, age, hire_date, salary from persons order by salary desc, hire_date;


分组子句：group by
如果SQL中使用了分组子句，那么在select子句里面，只能使用分组字段或者是统计函数。例如：
select age,count(age) from persons group by age;
注意：通常情况下分组子句会结合统计函数一起使用。

having条件子句
having子句通常是配合group子句一起使用，having将统计函数作为其条件字段。
select age,count(age) from persons group by age having count(age)>1;

limit子句：分页子句
limit 开始的行号（第一行为0）,显示几条记录
select * from persons limit 0,1;

distinct去重：
去除指定字段中的重复内容，只保留一个。
select distinct salary from persons;

列表名与表别名
列别名：给指定的列，起别名。可以使用as关键字。
select 列名 [as] 别名 from 表名
例如：
select pno as '编号' from persons;
表别名：给指定的表，起别名。使用as关键字修饰。


统计函数：
count():统计个数
max()：求最大值
min()：求最小值
avg()：求平均值
sum()：求和
注意，在使用统计函数时，如果没有配合group by子句使用，那么在select中只能输出统计函数的值，不能输出其他字段。例如：
select max(salary) from persons;
其中，max、min、avg、sum只能统计数据类型是数值型的字段，如果该字段值为null，那么统计函数将忽略该内容。我们可以通过ifnull(字段，替换值)，将null值转换为我们指定的值，进而参加相关的统计运算。
select avg(ifnull(salary,0)) from persons;

模糊查询：like
我们通常在查询时，如果条件内容不是非常明确的情况下，我们会使用like关键字进行模糊查询。通常会使用%、_来配合，其中%代表了0到n个字符，_代表了1个字符。
select * from persons where pname like '_i%';查找名字第二位是i的人员信息
select * from persons where pname like '%i%';查找名字中有i的人员信息

多表查询

1、查询分类
内连接（等值连接）
inner join 
select e.empno,e.ename,e.sal,e.deptno,d.dname from emp e inner join dept d on e.deptno=d.deptno where e.sal>2000;
等价于
select e.empno,e.ename,e.sal,e.deptno,d.dname from emp e, dept d where e.deptno=d.deptno and e.sal>2000;

外连接
分为左外连接与右外连接
left join on
right join on
何时使用外连接？
当进行两个表连接查询时，其中一方有不匹配记录，如果进行等值连接这些记录将不会被查询检索，如果我们需要查询时将匹配记录以及不匹配记录都查询出来，就需要使用外连接。那么不匹配记录存在的表位于连接语句的左侧，就使用左外连接，如果出现在右侧就使用右外连接。
例如：其中dept表中有不匹配的记录，deptno=40的记录。
左外连接
select e.empno,e.ename,d.deptno,d.dname from  dept d left join emp e on e.deptno=d.deptno;
右外连接
select e.empno,e.ename,d.deptno,d.dname from emp e right join dept d on e.deptno=d.deptno;

子查询
概念：将一个SQL查询语句的返回结果，作为另外一个查询语句的条件来使用。
分为单行子查询与多行子查询
单行子查询：作为条件返回的查询语句，其返回结果为一行记录。
多行子查询：作为条件返回的查询语句，其返回结果为多行记录。
子查询通常放在where条件子句中进行使用。但有时我们也可以将子查询作为虚表，放在from子句后面使用。但是这种使用方法，必须给子查询一个别名。例如：
select * from (select * from emp) as e;

MySQL数据库总结

2018-09-09

数据库语言

分类

数据库定义语言   DDL（create database/table，drop database/table，alter database/table）
数据库操纵语言   DML(insert、delete、update、select)
数据库控制语言   DCL（grant、revoke）
注意，数据库定义语言和数据库控制语言都是不可撤销的。而数据库操作语言是可以撤销 。

数据定义语言

1）、创建数据库
create database 数据库名；
2）、删除数据库
drop database 数据库名；
3）、打开数据库
use 数据库名；
4）创建数据表
create table 表名(列名 数据类型(长度),列名 数据类型（长度）,......) default charset=utf8
注意，如果创建表的时候，没有指定字符集，那么该表将默认采用其所在数据库的字符集。通常我们建议指定缺省的字符集。
5)、查看数据表的字符集情况
show variables like 'character%';
6）查看表结构
desc 表名

7）、数据库字段类型
字符类型：
char(长度)：定长的字符类型，一旦声明了长度，即使定义的字段数据不能达到最大长度，也会按照最大长度进行所占字节的分配。取值范围：0-255
varchar(长度)： 变长的字符类型，声明长度后，如果字段数据的长度达不到最大长度，则会根据实际长度给出数据的长度。取值范围：0-2^16-1
数值类型：
int(长度)：给出指定的字段数据长度。如果没有明确指定长度，则采用默认长度11。所占字节4个字节。

日期类型：
date：日期类型的字段，在定义时不需要指定长度。格式为YYYY-MM-DD
time：日期类型的字段，格式HH:MM:SS
datatime：日期类型的字段，格式YYYY-MM-DD HH:MM:SS
布尔型：
bool:布尔类型的字段，在定义时采用0代表假，1代表真。
实际在mysql数据库中，bool类型等同于数值型的tinyint(1)

8)、修改表结构
alter table 表名 [add/modify/change/drop] 字段名 [数据类型]
添加字段：
alter table tablename add 字段名 数据类型；
添加字段，位于表中的最后一列。
alter table tablename drop 字段名
删除一列，如果列中有数据，则全部被删除且不可恢复。
alter talbe tablename modify 字段名 数据类型
修改指定字段的数据类型及长度。
alter table tablename change oldcolumn newcolumn 数据类型
修改指定字段的名字
9）、修改表名
rename table 旧表名 to 新表名

数据操作语言

1）数据类型间的默认转换
字符串=>日期型
如果字段数据类型定义为日期型，例如：date（YYYY-MM-DD），如果我们队该字段插入数据时，采用'2018-08-23'的字符串格式进行数据插入，mysql会自动将字符串转为日期型。

字符串=>数值型
当字段数据类型为字符型时，但具体值为数值，我们使用该字段的值与数值进行运算，那么得到的结果为数值型。
同时，如果字段类型定义为数值型，我们可以将数值型字符串插入到指定的字段中。
如果，字段类型为字符型，其中由字符与数字混合组成（只要不是纯数字构成的字符串），此时与数值进行算数运算，那么该字段内容被转换为0来进行处理。
2）运算符
算数运算符
比较运算符
逻辑运算符
位运算符
3）插入数据
标准格式：
insert into 表名（字段1，字段2，字段3，....） values(字段1值，字段2值，字段3值，...)
insert into persons(pno,pname,age,hire_date,salary) values('10001','tom',21,'2018-03-12',3459);
通常，我们在声明表名的时候将需要插入数据的字段名一起进行声明，有利于提高插入的效率。同时如果有些字段值为null，可以在实际插入的时候省略。另外，字段值只需要与字段名称对应即可，不需要按照表中实际字段的位置进行填写。

insert into 表名 values(字段1值，字段2值，字段3值，...)
这种写法，需要我们在values的后面，按照表中各个字段的实际顺序进行一一对应的填写。同时，如果某个字段的值为null，也必须将null值进行标明。

插入多条语句：

insert into 表名（字段名） values(字段值1,...，字段值n)，(字段值1,...，字段值n)
一次性插入多条数据

insert into person_bak(pno,pname,age,hire_date,salary) select * from persons;
将指定表的查询结果，作为要插入到目标表中的数据，进行插入。要求是目标表结构与指定表的结构相同。
4）、更新数据
格式：
update 表名 set 字段1=新值，字段2=新值，...，字段n=新值 where 条件
进行更新操作时，一定要进行条件的约束，如果该语句没有条件子句的话，就表示对全表进行更新操作。
5）、删除数据
delete from 表名 where 条件
进行删除操作时，一定要添加条件，如果没有条件子句，就意味着删除全表内容。
6）、数据操作的撤销和提交
mysql数据库默认模式为自动提交：autocommit=on
我们如果需要对mysql进行撤销和提交操作，需要先将默认的自动提交模式取消，设置为off
即：set autocommit=off
当我们进行增删改操作时，如果执行rollback操作，那么该操作就会被撤销；
如果执行commit操作，那么该操作就会持久化到数据库中。
commit
rollback

数据库函数

1、字符串函数
length(字段)：返回该字段字符串的实际长度。
trim(字段)：将字符串左右两端的空格去除。
repeat(字段,n)：将字段中的字符串重复n次
replace(s,s1,s2)：使用s2替换字段s中的s1
substring(s,n,len):对字符串s的第n个位置开始，截取长度为len的子字符串。注意，此处字符串的字符位置从1开始。
2、数值函数
ceil()：返回数值的上限（只入不舍）
floor()：返回数值的下限（只舍不入）
mod(x,y)：返回两个数的余数
round(x,[y])：对x进行四舍五入操作，如果声明了y,则保留到小数点后y位
truncate(x,y)：对x进行截断操作，保留小数位y位
3、日期函数
curdate():当前日期年月日
curtime():当期时间时分秒
now():当前的时间 年月日 时分秒
year():返回当前的年份
month()：返回当前的月份
dayofmonth()：返回当前的天数
date_format():格式化指定字段的日期

4、系统函数
user():返回当前登录数据库的用户名

数据控制语言

1）创建用户
create user 用户名 identified by 密码
2）授予用户权限
grant 权限 on 数据库 to 用户名
grant all privileges on mydb.* to 'pythondb'@'localhost';
此处，将mydb数据库的所有权限都授予pythondb用户
同时，在授予权限时，可以选择授予不同的权限，如：查询、删除、修改、新增、修改表、创建表、删除表等等。
3）撤销用户权限
revoke 权限 on 数据库 from 用户名

将mydb数据库上的所有权限，从pythondb用户收回
4）删除用户
drop user 用户名
drop user 'pythondb'@'localhost';

删除表数据

delete
truncate
这两种方法都可以实现对表数据的删除，但是在实际操作中，他们的删除原理是不同的。
delete属于DML语句，所有的删除操作都是可以被撤销的。也就是意味着，进行的删除操作并没有真正对数据库的记录进行持久化删除，而且在数据库服务器中仍然保留了其记录数据的缓存，一旦执行rollback，就可以将已删除的内容进行恢复。
truncate删除数据，实际上是进行的表的截断操作，在删除记录的同时将这些记录在数据库服务器中的缓存同时进行删除。此操作不可撤销

事务

背景：银行转账过程中，如果发生了意外，需要使用事务行处理。
A账户有余额1000元，向余额为200元的B 账号进行转账500元，由于发生了某些故障，造成A账号在转账过程中出此案了错误，需要使用事务回滚（rollback）。如果A账号成功扣款，转账完成，B账号余额增加500元.
不使用事务产生的问题：
1、    当A 账户进行转账扣款时，发生错误。B账号并么有收到转账金额500元。而A账号已经提示完成扣款。
2、    当A 账号进行转账发生错误，未成功扣款，而B账号提示收到转账金额500元。
我们称上述的情况产生的数据为脏数据（未清洗de数据）。
使用事务处理的具体操作：
1、    关闭MySQL自动提交功能。 set autocommit= off 或者 0；
2、    开始事务：start transaction  / begin
3、    执行相应的sql操作
4、    提交或回滚: commit  rollback
5、    将MySQL自动：set autocommit =1;
概念：
在MySQL中事务主要用于处理数据量大，复杂度高的数据。只有定义了InnoDB引擎的数据库才支持事务。事务处理可以用来维护数据库的完整性，保证SQL操作要么全完成，要么全不执行。

事务满足四个特性: ACID
A：原子性。一个事物中的所有操纵，会作为一个整体来看待，在执行时如果有一个操作没有成功，那么所有操作都将被滚。回到事务开始前的状态。
C:一致性。在事务开始之前和事务结束之后，数据库的完整性没有被破坏，说明写入的资料符合数据库的所有设置的规则，包括数据的精度。
I:隔离性：数据库允许同时并发多个事务，并且同时对数据库进行读写操作，隔离性可以保证防止多个事务并发执行时由于交叉执行而导致数据的不一致。
事务的隔离分为：读未提交、读提交、可重复读、串行化
D：持久性。事务处理结束后，对数据的修改时永久佛如，即存储在数据库中。

MySQL数据库中myisam与innodb的区别
1、    MyISAM是非事务安全型的，而InnoDB是事务安全型的。
2、    MyISAM是锁支持到表级，而InnoDB支持行级。
3、    MyISAM是支持全文检索，而InnoDB是不利于全文检索的。
4、    MyISAM表保存为文件形式的，有利于跨平台时进行数据转移，而InnoDB比MyISAM更安全，可以保证表中数据不丢失的情况下，切换非事务到事务标
5、    如果在实际应用中，需要执行大量的select查询，那么MyISAM时优先选择的。
如果需要执行大量的增删改操作，那么InnoDB是优先选择的，可以提高多用户并发操作的性能。
通常，我们在实现构建数据库服务器时，采用读写分离的方式，即设置一台读服务器，一台写服务器，同时俩台服务器进行数据同步

网络原理

1、HTTP与HTTPS的区别
HTTP：我们称为超文本传输协议，主要用来实现从网络传输超文本到本地浏览器，实现高效且准确的超文本传送。目前HTTP的版本为1.1
HTTPS：我们认为就是以安全为目标的HTTP通道，即在HTTP协议的基础上加入了SSL层，简称HTTPS。
SSL的主要作用分为以下两部分：
第一、为HTTP传输文件提供安全的传输通道
第二、确认网站的真实性，通过HTTPS访问的网站都需要获取到CA认证。

2、网络七层模型（了解）
从下到上的顺序
物理层
数据链路层
网络层
传输层（TCP、UDP）
会话层
表示层
应用层（HTTP FTP TFTP SMTP SNMP DNS TELNET HTTPS POP3 DHCP）

3、HTTP的请求机制
第一，由客户端（浏览器）发出请求
第二，通过网络传输，到指定的服务器地址
第三，服务器接收相关请求，并进行解析
第四，服务器将最终的结果响应回客户端
第五，客户端接收到服务器响应后，通过浏览器解析，最终将内容进行展现

4、客户端访问服务器，发出请求，有四部分构成
第一，请求方法（request method），默认为get方式。只有在表单提交和文件上传时为post方法。
第二，请求网址（request url），即在浏览器中输入的网路地址。
第三，请求头（request header），用来标识为一个浏览器正常的访问
第四，请求体（request body），用来传输到服务器端的相关数据


5、请求方法中get与post的区别
第一、使用get方式提交请求时，相关数据会作为url的参数被一起进行提交，我们可以直接在地址栏中观察到相关数据。而采用post方式请求的url不会包含这些数据，而是直接通过表单提交到服务器。
第二、get方式提交请求，对于提交的数据大小不能大于1024字节，而post方式提交数据，没有大小的限制。
总之，如果提交的信息比较敏感或者字节较大时，需要使用post方式；如果只是普通的请求使用get方式。

二、HTML网页结构

面向对象

面向对象的概念

1、类：指具有相同属性或行为的一组事物。
对象：是指类的具体化。
类与对象的关系
类是一组对象的抽象表示，而对象是类的具体化表现。
2、类的语法结构
class 类名：
    变量声明语句
    执行语句
    def func(self):
        执行语句
其中，类名首字母要求大写（约定俗成），遵守Python命名规则，不能以数字开头，通常我们都是字母开头。
注意：
1）如果类中声明了函数，其函数的参数，必须有一个默认的参数为self，self代表了该类的实例化对象。
2）如果在类中声明了变量，且该变量位于函数体之外，这个变量我们成为类变量，类变量是公共的。类变量的调用可以直接使用类名进行调用，比如：
class Dog:  # 定义一个类，类名Animal
    color = '灰色'
    leg_num = 4
    weight = 20
    width = 150

    def wangwang(self):
        print('wangwangwangwang')

    def eat(self):
        print('一顿饭20块钱')

    def run(self):
        print('非常快')


husky = Dog()  # 将类实例化成对象，对象为dog
husky.run()
Dog.color

类变量，作为公共的变量，不管类实例化多少个对象，都只实例一份类变量。而且类变量会在对象实例化之前载入内容，一旦改变了类变量中的内容，所有对象中调用这个类变量的值都会随之改变。

3)在类实例化对象时，会将其定义的属性和行为，生成一份副本给具体的实例化对象。对象与对象之间，他们的属性和行为是不相互干扰的。

class Dog:  # 定义一个类，类名Animal
    color = '灰色'

    def __init__(self, num):
        self.num = num


    def getNum(self):
        print(self.num)


husky = Dog(5)  # 将类实例化成对象，对象为dog
samoye = Dog(3)
husky.getNum()
samoye.getNum()

我们在实例化Dog类时，生成两个对象husky、samoye，这两对象在实例化的过程中会生成两套成员变量和函数，他们的相互不干扰的。我们可以观察上面的代码，发现num变量属于成员变量，即husky和samoye各有持有相同的成员变量，但是修改num的值，husky为5，而samoye为3。结果表明他们的成员变量只能归各自的对象所拥有，并进行改变。对象与对象之间不会相互影响。
4）使用构造函数，来初始化成员变量
class Person:

    def __init__(self):  # 构造函数，在实例化对象时被默认调用，用来给对象的成员变量赋值
        self.name = 'tom'
        self.age = 21
        self.tel = '1350000000'
        self.address = '北京'
        self.sex = '男'

    def setName(self, name):
        self.name = name

        return self.name

使用__init__()函数，用来初始化成员变量。对象在实例化时，会默认的调用__init__()函数，同时将成员变量进行初始化。同时，在构造函数中，使用self.变量名进行初始化的成员变量，都是公有的。即在类外可以通过实例化的对象调用。

类变量与成员变量的区别
1、类变量是声明在类中，位于函数之外声明的变量。这些变量时公共的，即在类外使用类名.类变量的方式进行调用。我们一旦修改类变量的值，所有调用类变量的内容均发生改变。
2、成员变量，是声明在类中的构造函数中，通过self.变量来进行声明赋值的。这些成员变量是公有的，即在类外可以通过对象名.成员变量的方式进行调用。所有的对象都有自己的成员变量，也就意味着每个对象的成员变量都是独立的，不会相互干扰。
3、类变量是该类所有实例化对象所公有的，它会早于对象实例化之前被载入内容中。
成员变量都是跟随对象实例化的过程中，被声明出来的。它是伴随着对象的产生而产生，消亡而消亡的。
4、成员变量和方法，默认情况下都是公有的。也就是说在类外可以直接使用对象进行调用。我们可以通过为成员变量设置双下划线，将其标识为私有的成员变量，例如：self.__name = 'tom'。私有的成员变量只能在类内进行直接使用，类外必须通过相关方法进行间接调用。

Python中对象创建的流程

1、Python创建一个对象，首先要调用__new__方法，用来创建并返回一个对象。通常，Python中不需要我们直接显示调用，它会默认调用object.__new__()方法
2、然后，调用__init__方法，对该对象进行初始化。

析构函数：
1、析构函数与构造函数一样，都是Python中特殊的函数。
2、析构函数如果用户没有定义的话，Python会提供一个默认的析构函数
3、析构函数__del__(self)，作用用来释放掉对象所占用的内存空间，在Python回收对象空间之前自动被执行。
4、通常在析构函数中，主要是将对象中占用大量资源的变量进行手动的释放，便于后续Python解释器去回收这些内存空间。

继承

1）Python中能够实现单继承与多继承。
单继承：就是子类只有一个直接父类。
多继承：子类有两个及两个以上的直接父类。
2）子类继承父类，子类可以继承父类的相关特征。子类继承了父类的方法，如果在多继承情况下，父类出现同名函数，那么子类继承该函数时，根据继承的顺序进行调用。
3）super()函数，它指代了当前类的父类。我们可以在子类中调用super函数，用来初始化父类。但是使用super()调用父类的构造函数时，只能对多继承中的第一个父类生效。
4)抽象与继承的区别与关系
抽象：指对同一类别事物的相似部分进行抽取。抽象我们认为是对具体化对象的相关行为和属性的泛化过程。
继承：Python中我们通常认为类就是一个抽象的内容。类本身并不能直接执行，需要通过继承，进行对象的实例化，进而获得具体的对象本身，才能执行相关的功能。
也就是说继承其实是抽象的具体化过程。

5）多继承中，当一个子类继承多个父类时，我们需要如下声明：
class SubClass(A,B):
其中，A、B是SubClass的父类。那么，如果我们分为两种个情况：
第一，子类中没有显式的定义__init__()，那么在实例化子类时调用其父类的构造函数。如果有多个类，都定义了__init()函数，那么它会调用第一个具有__init__()函数的父类。
第二，子类中显式的定义理论__init_(),那么实例化子类时不再调用父类的构造函数。此时，如果我们要调用父类的构造函数，有两种方法。
1）在子类的构造函数中，显式的调用对应父类的构造函数，例如：
A.__init__()
2)采用super()函数，super().__init__()或者super(子类，self).__init__()
但是如果遇到其多个父类，他们的构造函数同名时，我们根据他们在MRO中的顺序，分别调用他们，例如：
class SubClass(B, A):
    def __init__(self):
        super().__init__()   # 此处调用了父类B
        super(B, self).__init__() #此处调用了父类C
        print('我是子类')
注意，在这里不要看到super就认为是父类，但是在多继承时，实际上super指的是MRO中的下一个类。

9）classmethod方法
在Python中，我们通常使用classmethod用来修饰一个方法，这个方法就被称为类方法，类方法通常用来操作类变量。类方法和类变量一样，都需要通过类名来调用。

10)staticmethod方法
在Python中，我们使用staticmethod来修饰函数，该函数被称为静态方法，首先，静态方法不需要使用对象参数或类参数作为默认参数。其次，静态方法通常只能调用类变量，而不可以使用成员变量。最后，在类外进行调用时，可以使用类名来调用。

实例方法、类方法与静态方法的区别
实例方法：第一个参数必须是实例对象（self）,通过它来传递实例的属性和方法，只能由实例对象来调用。
类方法：使用装饰器@classmethod，第一个参数必须是类对象（cls）,通过它来传递类的属性和方法。可以通过实例对象或类对象来调用，通常使用类对象调用。
静态方法：使用装饰器@staticmethod,没有默认参数（self和cls），但是方法体中不能使用类或实例对象的任何属性和方法。可以通过实例对象或类对象进行调用。

1、类方法和静态方法都可以通过对象实例或者类名来调用。
2、类方法主要针对的是类，可以通过继承和重新定义
3、静态方法不能被继承，无法被重写，可以被认为是全局函数。静态方法在使用时完全不会用到对象实例本身。静态方法逻辑上属于类，但是不会涉及到类中的属性和方法的操作。
